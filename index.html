A tiny browser bridge for Fran’s tweet: quote Maya Protocol liquidity, send one ETH mainnet transaction with the swap memo, receive native ZEC to transparent, unified, or shielded addresses. No wrapped asset theater.

Important: Maya now accepts shielded/unified ZEC destinations on supported routes, but the address is still embedded in the swap memo on Ethereum. For best privacy, use a fresh shielded/unified address and avoid reusing it.

How it works

Maya nodes observe your Ethereum transaction to the current vault. The tx data contains the memo: swap to ZEC and pay your Zcash address. If the memo/route is invalid, Maya refunds according to protocol rules.

This is intentionally 0 Solidity: ethskills says don’t put logic onchain unless it needs to be there. Here the correct move is a safe browser flow over an existing native-swap protocol.
